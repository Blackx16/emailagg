## Authentication

{% if auth_description is defined %}
{{ auth_description }}
{% else %}
All API endpoints require authentication. Two authentication methods are supported:

### JWT Bearer Token (External API)
```
Authorization: Bearer <jwt_token>
```
Tokens are issued via the Telegram WebApp login flow and validated by the `basic_auth_middleware`.

### Telegram WebApp Signature (Mini App)
```
X-Telegram-Auth: <initData>
```
The backend verifies the HMAC-SHA256 signature using the bot token as the key.
{% endif %}

### `GET /api/accounts`
Returns all connected mail accounts for the authenticated user.

**Response:**
```json
{
  "accounts": [
    {
      "id": "uuid",
      "email": "user@example.com",
      "provider": "gmail",
      "status": "active",
      "last_sync": "2026-07-04T10:00:00Z"
    }
  ]
}
```

---

### `POST /api/accounts/connect`
Initiates OAuth connection for a mail provider.

**Request:**
```json
{
  "provider": "gmail",
  "redirect_uri": "https://app.teleswift.io/oauth/callback"
}
```

**Response:**
```json
{
  "oauth_url": "https://accounts.google.com/o/oauth2/auth?...",
  "state": "csrf_token"
}
```

---

### `GET /api/rules`
Lists all active forwarding/filtering rules for the authenticated user.

### `POST /api/rules`

```json
  "account_id": null
}
```

## Error Responses

All errors follow a consistent JSON structure:

```json
{
  "detail": "Human-readable error message",
  "code": "ERROR_CODE",
  "field": "affected_field_name"
}
```

| Status Code | Meaning |
|-------------|---------|
| 400 | Bad Request — Invalid input |
| 401 | Unauthorized — Missing or invalid auth |
| 403 | Forbidden — Insufficient permissions |
| 404 | Not Found |
| 429 | Too Many Requests — Rate limit exceeded |
| 500 | Internal Server Error |
